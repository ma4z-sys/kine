Overview

Kine is a lightweight command-line tool that lets you run Android APKs natively on Linux. It wraps Waydroid — a real Android container using LXC and the host Linux kernel — giving you near-native performance without a full emulator UI.

Each app opens in its own window. No emulator dashboard. No fake filesystem. Just your APK running directly against real Android runtime.

Real Android runtime
Native window per app
Wayland + X11 support
GPU acceleration
Auto display detection
Smart error recovery

Installation

Clone the repo and run the installer. It sets up the kine binary, sudoers rules, and systemd service.

bash 
# Clone kine
git clone https://github.com/yourname/kine.git
cd kine

# Install system-wide (requires sudo)
sudo bash install.sh
What the installer does: copies files to /opt/kine/, creates /usr/local/bin/kine symlink, configures sudoers for passwordless Waydroid commands, and registers the waydroid-container systemd service.

Manual dependency install (if needed)

bash 
# Add Waydroid repository
curl https://repo.waydro.id | sudo bash

# Install core packages
sudo apt install -y waydroid lxc lxc-utils aapt xwayland

# Load kernel modules (run once)
sudo modprobe binder_linux num_devices=254
sudo modprobe ashmem_linux   # (older kernels only)

# Make modules load at boot
echo "binder_linux" | sudo tee /etc/modules-load.d/waydroid.conf

Launch Flow

Complete flow from fresh system to running app:

1
sudo bash install.sh
Installs kine binary, configures sudoers and systemd service
2
kine init pixel5
Checks deps → loads kernel modules → detects display server → downloads Android image → starts container
3
kine launch myapp.apk
Validates APK → ensures session active → installs if needed → launches in dedicated window
✓
App is running
Native window, no emulator dashboard, keyboard/mouse/audio all working
$
Commands
kine init <model>
Required first
Initialize the Android runtime. Checks dependencies, loads kernel modules, detects display server, downloads Android image, and starts the container. <model> is a profile name like pixel5, pixel4, or samsung.
kine launch <app.apk>
Main command
Validates the APK, ensures the Android session is running, installs if not already installed, and launches the app in its own window. Auto-recovers from display errors.
kine install <app.apk>
Optional
Install an APK into the Android runtime without launching it. Useful for pre-loading apps or batch installs.
kine stop
Lifecycle
Stop the Android session and container. Frees RAM and background processes. Re-run kine launch to restart automatically.
kine status
Info
Show current state: initialization status, container health, display backend, GPU mode, and list of installed apps.
kine logs
Debug
Tail the kine activity log at ~/.kine/logs/kine.log. All commands, errors, and display events are recorded here.

Configuration

Edit ~/.kine/config/kine.conf to customize behavior:

bash — ~/.kine/config/kine.conf 
# Android image type
KINE_CHANNEL=lineage        # or "vanilla" (no GApps)

# Display backend: auto, wayland, or x11
KINE_DISPLAY_BACKEND=auto

# Force software rendering if GPU causes issues
KINE_FORCE_SOFTWARE_RENDER=false

# Networking mode
KINE_NETWORK_MODE=nat       # or "bridge"

# Audio backend
KINE_AUDIO_BACKEND=auto     # or "pulseaudio" / "pipewire"

# Session startup timeout (seconds)
KINE_SESSION_TIMEOUT=30

# Keep APK copies in ~/.kine/apps/
KINE_KEEP_APKS=true

Display Server

Kine auto-detects your display server and configures Waydroid accordingly. It handles the most common GNOME/Wayland/X11 edge cases automatically.

WAYLAND (recommended)Best experience
# Set automatically by kine
export XDG_SESSION_TYPE=wayland
export WAYLAND_DISPLAY=wayland-0
export XDG_RUNTIME_DIR=/run/user/$UID
X11 / XWAYLAND FALLBACKAuto-fallback
# Set automatically by kine on X11
xhost +local:
export DISPLAY=:0
unset WAYLAND_DISPLAY
GNOME + Wayland issue? If you get "Failed to connect to display", kine automatically tries: restart container → switch to XWayland → reinitialize session. All silently before showing any error.

Troubleshooting
ERROR: Failed to connect to display

Waydroid can't find the display socket. Usually caused by display server mismatch.

# Reset display config and reinitialize
rm ~/.kine/config/display.env
kine init pixel5

# Or manually force X11 mode
export DISPLAY=:0 WAYLAND_DISPLAY=
xhost +local:
ERROR: Session could not be started

Container failed to start or session hung.

# Check Waydroid logs
sudo waydroid log

# Restart container
sudo systemctl restart waydroid-container
waydroid session start
ERROR: binder module not found

Kernel lacks binder support. Install extra modules.

sudo apt install linux-modules-extra-$(uname -r)
sudo modprobe binder_linux num_devices=254
App installs but won't open

Package name couldn't be resolved. Install aapt for automatic detection, or launch manually.

# Install aapt for package name resolution
sudo apt install -y aapt

# List installed packages
waydroid app list

# Launch by package name
waydroid app launch com.example.myapp
